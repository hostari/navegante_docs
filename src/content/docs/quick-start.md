### 1. Prerequisites

To use start using Navegante, you need to have:

1. An active Github Account\*
2. Public or private repositories with the code that you want to deploy
3. Optional: A Docker Compose\*\* file

_\*We only support Github for now._

_\*\*We also support projects with `docker-compose.yml` files. Click [ here ] to learn more._

### 2. Create your free Navegante Account

Creating a Navegante account is free and easy to do. Click [here](https://navegante.app/account_registrations/onboarding?utm_source=nvg_docsite) to create your account.

All Navegante accounts are entitled to 1 free Infrastructure Project (Hobby Plan). Learn more about our free-forever Hobby plan [ here ].

### 3. Install the Navegante Github App

Navegante relies on Github to securely pull, build, and deploy your software to the Internet.

After creating your account, you will be prompted to install the Github App and provide **read-only access** to your desired personal or organization repos.

Read the [Github Integration](architecture/github-integration) page to learn more about how we securely handle your Github repos.

### 4. Create your first Infrastructure Project

Infrastructure projects are collections made up of Github repos that you want to deploy.

To create an Infrastructure Project:

1. Click on the "Create a New Infrastructure Project" button.
2. Write the desired name of your Infrastructure Project (or leave it blank for an auto-generated name).
3. Select the desired Compute Plan for your Infrastructure Project.
4. Select the repos you want to add to your Infrastructure Project.
5. Create and view your project in the dashboard.

Infrastructure Projects are billed according to the Compute Plan you selected during the project creation process. If you wish to update your Compute Plan, reach out to us at `support@navegante.app`.
